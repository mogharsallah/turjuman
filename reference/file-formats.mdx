Kalem stores all translation values internally as [ICU MessageFormat](https://unicode-org.github.io/icu/userguide/format_parse/messages/) strings. When you run `kalem pull`, the CLI converts each value from ICU into the native representation expected by the target format. When you run `kalem push`, the CLI converts in the opposite direction — reading the format's native representation and writing ICU back to Kalem.

---

## Supported formats

| Format ID | File type | Notes |
|---|---|---|
| `json-nested` | `.json` | i18next-style nested objects; keys are dot-separated paths |
| `json-flat` | `.json` | Flat object; keys are stored literally (dots are part of the key name) |
| `yaml` | `.yaml` / `.yml` | Same nesting semantics as `json-nested` |
| `arb` | `.arb` | Flutter Application Resource Bundle; carries descriptions in `@key` metadata |
| `properties` | `.properties` | Java resource bundles; descriptions are written as `# comment` lines above each key |
| `csv` | `.csv` | Two-column (key, value) with an optional description column |
| `android` | `strings.xml` | Android resource file; includes `<plurals>` support |
| `ios-strings` | `.strings` | iOS singular strings; descriptions are written as `/* comment */` above each entry |
| `ios-stringsdict` | `.stringsdict` | iOS plurals using `NSStringPluralRuleType` |

Run `kalem formats` to get the live list directly from your Kalem instance.

<Note>
For iOS projects you should configure **both** `ios-strings` and `ios-stringsdict` as separate targets in your `kalem.config.json`, both pointing at the same `.lproj` directory. Singular keys are handled by `.strings` and plural keys are handled by `.stringsdict` — they complement each other rather than overlap.
</Note>

## Plural handling

Kalem stores plurals as ICU MessageFormat plural expressions:

```
{count, plural, one {# item} other {# items}}
```

Each format adapter converts this representation to and from the format's native plural structure:

### Android (`strings.xml`)

Plurals are converted to and from `<plurals>` elements with `<item quantity="...">` children. Android supports the plural categories `zero`, `one`, `two`, `few`, `many`, and `other`.

```xml
<plurals name="item_count">
    <item quantity="one">%d item</item>
    <item quantity="other">%d items</item>
</plurals>
```

### iOS (`.stringsdict`)

Plurals are converted to and from `.stringsdict` property list entries using `NSStringPluralRuleType` keys. Each ICU plural category maps to the corresponding `NSStringPluralRuleType` value.

```xml
<key>item_count</key>
<dict>
    <key>NSStringLocalizedFormatKey</key>
    <string>%#@items@</string>
    <key>items</key>
    <dict>
        <key>NSStringFormatSpecTypeKey</key>
        <string>NSStringPluralRuleType</string>
        <key>NSStringFormatValueTypeKey</key>
        <string>d</string>
        <key>one</key>
        <string>%d item</string>
        <key>other</key>
        <string>%d items</string>
    </dict>
</dict>
```

### Flutter ARB (`.arb`)

ARB files use ICU MessageFormat natively, so no conversion is needed for plural expressions. The ICU string is written directly into the ARB value.

```json
{
  "itemCount": "{count, plural, one {# item} other {# items}}",
  "@itemCount": {
    "description": "Number of items in the list"
  }
}
```

## Path patterns

The `path` field in each `kalem.config.json` target uses `{locale}` as a placeholder. The CLI substitutes it with each locale code when reading or writing files.

```json
{ "format": "json-nested", "path": "locales/{locale}.json" }
```

For `fr`, this resolves to `locales/fr.json`. For `zh-TW`, it resolves to `locales/zh-TW.json`.

<Note>
Support for gettext PO (`.po`) and XLIFF (`.xliff` / `.xlf`) format adapters is planned for a future release.
</Note>
