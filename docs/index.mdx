**Turjuman is a lightweight, open-source alternative to commercial translation-management SaaS.**
Instead of a heavy web dashboard, you manage projects, keys, locales and
translations through an **MCP server** connected to Claude Code (or any MCP client/agent). A thin
**developer CLI** handles the deterministic file work — pulling and pushing locale files in your
repo and CI.

It runs on a **serverless AWS stack (Lambda + DynamoDB)** that costs next to nothing to host and
fits comfortably in the free tier. No always-on servers, no Cognito.

```
Claude Code / agent ──Streamable HTTP + API key──┐
Developer CLI / CI ──REST + API key──────────────┤
                                                 ▼
                          Lambda Function URLs ──► DynamoDB (single table)
```

## Why MCP-first?

Translation management is mostly conversation and judgement: "add a French locale", "translate the
untranslated checkout strings", "what does `cart.empty` say in German?". An agent with the right
tools does this naturally. The connected LLM **does the translating itself** — there is no separate
machine-translation engine to pay for or configure.

The CLI exists for the things you should *not* hand to an LLM: deterministically exporting a
JSON/YAML file, downloading translations, and syncing them in CI.

## How the code is organized

A pure shared core (`@turjuman/schema`) plus the server brain (`@turjuman/core`) wrapped by two thin
transports. The transports only translate HTTP ↔ service calls — **all business logic and all
authorization live in the core packages** — so the MCP and REST surfaces can never drift in
behaviour or permissions.

| Package | What it is |
|---|---|
| `packages/schema` | The pure, AWS-free base: domain model, zod schemas, RBAC policy, ICU helpers, QA engine (the shared brain) |
| `packages/formats` | Localization file-format adapters (JSON/YAML/ARB/CSV/Android/iOS) over a canonical ICU representation |
| `packages/core` | Server core: the single-table DynamoDB repository + services, built on `@turjuman/schema` |
| `packages/mcp-server` | Stateless Streamable HTTP MCP server → Lambda (ships a pre-bundled Lambda asset) |
| `packages/api` | REST API for the CLI/CI → Lambda, plus the webhook dispatcher (ships pre-bundled Lambda assets) |
| `packages/cli` | The lightweight `turjuman` developer CLI (no AWS SDK) |
| `packages/aws-cdk` | The `@turjuman/aws-cdk` CDK construct — the composable deploy topology |
| `packages/aws-deploy` | The `turjuman-aws-deploy` self-host CLI (deploys the construct, self-bootstraps) |
