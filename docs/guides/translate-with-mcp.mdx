The MCP server is Kalem's primary interface. Connect any MCP-compatible agent — Claude Code, a custom agent loop, or any other client — and drive your entire translation workflow in natural language. The server exposes 40+ tools covering projects, keys, translations, glossaries, translation memory, QA, members, and API keys.

## Connect your MCP client

Add the following to your `.mcp.json` (or equivalent client config) to register Kalem as an MCP server:

```json
{
  "mcpServers": {
    "kalem": {
      "type": "http",
      "url": "https://<your-mcp-function-url>/",
      "headers": { "Authorization": "Bearer <your-api-key>" }
    }
  }
}
```

Replace `<your-mcp-function-url>` with your self-hosted MCP endpoint and `<your-api-key>` with a valid API key.

<Tip>
  Scope the toolset your agent can see by appending `?tools=` or `?groups=` query parameters to the URL. For example, `?groups=read,translations` exposes only read tools and translation-management tools — useful for restricting a reviewer agent to the capabilities it actually needs.
</Tip>

<Warning>
  Your API key grants full access to your Kalem instance. Never commit it to version control. Use an environment variable or a secrets manager, and reference it from your MCP config at runtime.
</Warning>

## Example workflows

<AccordionGroup>
  <Accordion title="Stand up a project and translate it">
    **Prompt your agent:**

    > "Create a project called 'Marketing Site', add English as the base locale and French and German as targets, then translate all strings."

    **What the agent calls:**

    1. `create_project` — creates the project and returns a `projectId`
    2. `add_locale` — adds `en` (base), `fr`, and `de`
    3. `bulk_create_keys` — imports your source strings
    4. `bulk_set_translations` — sets base-locale values
    5. `translate_keys` — triggers AI translation into `fr` and `de`
  </Accordion>

  <Accordion title="Run a review pass">
    **Prompt your agent:**

    > "Review all French translations in the 'Marketing Site' project and flag any that sound unnatural or don't match our tone."

    **What the agent calls:**

    1. `list_untranslated` — finds strings still needing attention
    2. `list_stale` — surfaces strings whose source has changed since last translation
    3. `get_translation` — fetches individual values for inspection
    4. `set_translation` — updates values the agent corrects
    5. `run_qa_checks` — validates the final state before sign-off
  </Accordion>

  <Accordion title="Translate with glossary and memory">
    **Prompt your agent:**

    > "Translate the new keys in the 'Dashboard' namespace into Spanish, respecting our glossary and reusing any existing approved translations."

    **What the agent calls:**

    1. `list_glossary` — retrieves your glossary terms so the agent can enforce consistent terminology
    2. `lookup_translation_memory` — checks for previously approved translations that match the source string
    3. `bulk_set_translations` — writes new translations, pre-populated from memory hits
    4. `translate_keys` — fills remaining strings using AI, informed by the glossary context
  </Accordion>

  <Accordion title="Grant access to a teammate">
    **Prompt your agent:**

    > "Add alice@example.com as a translator and generate a scoped API key she can use with the CLI."

    **What the agent calls:**

    1. `add_member` — invites the user and assigns the `translator` role
    2. `create_api_key` — creates a key with a meaningful description (e.g. `"alice CLI key"`) and appropriate scope
  </Accordion>

  <Accordion title="Revoke a leaked key">
    **Prompt your agent:**

    > "One of our API keys was leaked in a log file. Find it and revoke it."

    **What the agent calls:**

    1. `list_api_keys` — returns all keys with their descriptions and metadata so you can identify the compromised one
    2. `revoke_api_key` — permanently revokes the key by ID
  </Accordion>
</AccordionGroup>

## Tips for working with the MCP server

**Give keys meaningful descriptions.** When you call `create_api_key`, always set a `description` like `"CI pipeline – main branch"` or `"Alice CLI key"`. It makes `list_api_keys` output actionable and makes revocation straightforward.

**`bulk_set_translations` skips unknown keys.** If you pass a key name that doesn't exist in the project, the tool skips it and reports it in the response rather than failing the whole batch. Check the response for a `skipped` array if you suspect a mismatch.

**Paginate large result sets.** Tools like `list_keys`, `list_untranslated`, and `list_stale` accept a `limit` parameter and return a `nextCursor` when there are more results. Pass that cursor back as `cursor` in the next call to page through the full set.

**The server is stateless.** There is no session to manage and no state that persists between tool calls beyond what's stored in Kalem itself. Your agent can call tools in any order, retry failed calls freely, and reconnect without ceremony.
