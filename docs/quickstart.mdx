---
title: Quickstart
description: "Run the entire Turjuman stack locally against LocalStack with one hot-reloading command, and connect an MCP client in minutes."
---

The fastest way to see Turjuman work is to run everything against a local
[LocalStack](https://www.localstack.cloud/) (it backs the dev loop and the test tiers alike). You
need [Docker](https://docs.docker.com/get-docker/) and Node 20.12+; no cloud account is needed.

<Steps>
  <Step title="Install and build">
    ```bash
    npm install && npm run build
    ```
  </Step>
  <Step title="Start LocalStack">
    ```bash
    npm run stack:up
    ```

    This starts the shared LocalStack container on `:4566` and waits for it to be healthy.
  </Step>
  <Step title="Configure your environment">
    Copy the example env file — it points the dev servers at LocalStack with dummy AWS credentials:

    ```bash
    cp .env.example .env
    ```
  </Step>
  <Step title="Create the table and your first owner">
    This creates the table and the first `OWNER`, and prints your API key **once**:

    ```bash
    npm run dev:setup you@example.com "You"
    ```

    Copy the printed key — it is shown only this once.
  </Step>
  <Step title="Start both transports">
    One command runs the MCP server and REST API with hot reload from source:

    ```bash
    npm run dev   # MCP on http://localhost:3000, REST on http://localhost:4000
    ```

    Edit anything under `packages/` and both servers reload automatically.
  </Step>
  <Step title="Connect your MCP client">
    Point your MCP client (e.g. Claude Code) at `http://localhost:3000/` with the printed API key:

    ```json
    {
      "mcpServers": {
        "turjuman": {
          "type": "http",
          "url": "http://localhost:3000/",
          "headers": { "Authorization": "Bearer <your-api-key>" }
        }
      }
    }
    ```

    Then just ask: *"Create a project called Web App with base locale en, add fr and es, and
    translate everything into French."*
  </Step>
</Steps>

<Note>
  The same API key both authenticates you and determines your permissions. See
  [Roles & permissions](/concepts/roles-and-permissions) for how to scope keys to translators
  and CI.
</Note>
