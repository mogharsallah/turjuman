This is the fastest path from zero to a running Turjuman instance: deploy to your AWS account, connect your AI agent, and translate a project in natural language. Each step links to the full detail so this page stays short.

<Note>
  Want to evaluate Turjuman first without an AWS account? [Try it locally](/guides/try-it-locally) runs the whole stack on your machine with LocalStack, then come back here when you're ready to deploy for real.
</Note>

## Before you start

You need an AWS account with credentials configured and Node.js 24 or later. See [Self-hosting → Overview](/self-hosting/overview) for the full prerequisites and what you take on as the operator.

## Deploy and translate

<Steps>
  <Step title="Deploy to AWS">
    Install the construct into a tiny CDK app, then deploy. It vendors its Lambda assets, so there's no SAM CLI and no repo clone:

    ```bash
    npm install @turjuman/aws-cdk aws-cdk-lib constructs
    cdk bootstrap   # once per account+region
    cdk deploy      # prints McpUrl / ApiUrl / TableName
    ```

    See [Deploy to AWS](/self-hosting/deploy) for the `app.mjs` setup, stack outputs, and updates.
  </Step>

  <Step title="Create your first owner">
    Claim ownership over HTTP and capture your API key — printed **once**, and saved to `~/.turjuman/auth.json`:

    ```bash
    turjuman bootstrap --url <ApiUrl> --email you@example.com --name "Your Name"
    ```

    <Warning>
      The bootstrap route is unauthenticated until the first owner is created — run it immediately, before sharing the `ApiUrl`. See [Security & API keys](/self-hosting/security).
    </Warning>
  </Step>

  <Step title="Connect your MCP client">
    Add your `McpUrl` and API key to your client's MCP config. For Claude Code, create or update `.mcp.json` in your project root:

    ```json .mcp.json
    {
      "mcpServers": {
        "turjuman": {
          "type": "http",
          "url": "<McpUrl>",
          "headers": {
            "Authorization": "Bearer <your-api-key>"
          }
        }
      }
    }
    ```

    Restart your client to pick up the change. For other clients, see [Connect Claude Code](/guides/connect-claude-code).

    <Warning>
      Your API key grants full access to your instance. Keep it out of version control — reference it via an environment variable or a secrets manager rather than committing the literal value.
    </Warning>
  </Step>

  <Step title="Translate your first project">
    With the MCP server connected, ask your agent to set up and translate a project:

    > *"Create a project called Web App with base locale `en`. Add `fr` and `es` locales. Then translate all the keys into French and Spanish."*

    Turjuman's MCP tools handle the project setup, and the connected LLM does the translating — there is no separate machine-translation engine. For the full set of workflows, see [Translate with MCP](/guides/translate-with-mcp).
  </Step>
</Steps>
