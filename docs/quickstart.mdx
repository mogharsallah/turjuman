---
title: Quickstart
description: "Run the entire Turjuman stack locally against LocalStack with one hot-reloading command, and connect an MCP client in minutes."
---

The fastest way to see Turjuman work is to run everything against a local
[LocalStack](https://www.localstack.cloud/) (it backs the dev loop and the test tiers alike). You
need [Docker](https://docs.docker.com/get-docker/) and Node 24+; no cloud account is needed.
Turjuman uses [pnpm](https://pnpm.io) (pinned via `package.json`'s `packageManager` field) — run
`corepack enable` once to activate it.

<Steps>
  <Step title="Install and build">
    ```bash
    pnpm install && pnpm run build
    ```
  </Step>
  <Step title="Start LocalStack">
    ```bash
    pnpm run localstack:up
    ```

    This starts the shared LocalStack container on `:4566` and waits for it to be healthy.
  </Step>
  <Step title="Configure your environment">
    Copy the example env file — it points the toolchain at LocalStack with dummy AWS credentials:

    ```bash
    cp .env.example .env
    ```
  </Step>
  <Step title="Run Turjuman">
    One command deploys the stack into LocalStack, hot-reloads your edits, and **prints the MCP and
    REST URLs plus a fresh API key**:

    ```bash
    pnpm run dev
    ```

    Copy the printed `apiKey` and `MCP` URL. Leave it running — edits under `packages/` are served on
    the next request without a redeploy.
  </Step>
  <Step title="Connect your MCP client">
    Point your MCP client (e.g. Claude Code) at the printed **MCP URL** with the printed API key:

    ```json
    {
      "mcpServers": {
        "turjuman": {
          "type": "http",
          "url": "<MCP URL from pnpm run dev>",
          "headers": { "Authorization": "Bearer <your-api-key>" }
        }
      }
    }
    ```

    Then just ask: *"Create a project called Web App with base locale en, add fr and es, and
    translate everything into French."*
  </Step>
</Steps>

<Note>
  The same API key both authenticates you and determines your permissions. See
  [Roles & permissions](/concepts/roles-and-permissions) for how to scope keys to translators
  and CI.
</Note>
