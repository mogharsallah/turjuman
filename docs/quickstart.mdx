title: "Get Kalem running locally in minutes with LocalStack"

This guide gets you a fully working Kalem stack running on your machine in minutes. Instead of deploying to AWS, you run the MCP server and REST API inside LocalStack — a local AWS emulator — using Docker. No cloud account, no credentials, no cost — just Node.js, pnpm, and Docker.

<Note>
  The API key printed by the setup script determines both your identity and your permissions. Whoever holds it has full OWNER access. Store it somewhere safe and keep it out of version control.
</Note>

## Prerequisites

- Node.js 24 or later
- pnpm (enabled via `corepack enable`)
- Docker (for LocalStack)
- The Kalem repo cloned locally

## Steps

<Steps>
  <Step title="Install dependencies and build">
    Clone the repo (if you haven't already) and run the standard install + build:

    ```bash
    pnpm install && pnpm run build
    ```
  </Step>

  <Step title="Start LocalStack">
    Start the local AWS emulator (LocalStack) using Docker Compose. This provides DynamoDB, Lambda, and the other AWS services Kalem needs locally:

    ```bash
    pnpm run localstack:up
    ```

    This starts LocalStack on port 4566 and waits until it is healthy before returning.
  </Step>

  <Step title="Start the dev stack">
    Deploy Kalem into LocalStack and bootstrap your first owner in one command:

    ```bash
    pnpm run dev
    ```

    This builds and deploys the MCP and API Lambda functions into LocalStack with hot-reload, creates a first OWNER user, and prints your stack URLs and API key:

    ```
    Deployed "kalem-dev".
      MCP:     http://localhost:4566/restapis/.../mcp/...
      REST:    http://localhost:4566/restapis/.../api/...
      owner:   dev-owner@kalem.test
      apiKey:  km_live_xxxxxxxxxxxxxxxxxxxx

    Use the key as `Authorization: Bearer <apiKey>`.
    ```

    Copy the `MCP` URL and `apiKey` values — you will need them in the next step. The process stays running and hot-reloads code changes. Press `Ctrl+C` to stop.
  </Step>

  <Step title="Connect your MCP client">
    Add Kalem to your MCP client configuration. For Claude Code (`.mcp.json` in your project root):

    ```json
    {
      "mcpServers": {
        "kalem": {
          "type": "http",
          "url": "<MCP URL printed by pnpm run dev>",
          "headers": {
            "Authorization": "Bearer <your-api-key>"
          }
        }
      }
    }
    ```

    Replace `<MCP URL printed by pnpm run dev>` and `<your-api-key>` with the values printed in the previous step. Restart your MCP client to pick up the new config.

    <Note>
      Keep your API key out of version control. If `.mcp.json` is committed, reference the key via an environment variable or a secrets manager rather than pasting the literal value. Revoke any leaked key immediately using the `revoke_api_key` MCP tool.
    </Note>
  </Step>

  <Step title="Try it">
    With the MCP server connected, ask your agent to get started:

    > *"Create a project called Web App with base locale `en`. Add `fr` and `es` locales. Then translate all the keys into French and Spanish."*

    Kalem's MCP tools handle the project setup, and the connected LLM translates the keys directly — no separate translation engine needed.

    You can also ask things like:
    - *"What projects do I have?"*
    - *"Add a key called `checkout.empty_cart` with the English value `Your cart is empty.`"*
    - *"Run QA on the French locale and show me any issues."*
  </Step>
</Steps>
