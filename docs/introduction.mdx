Turjuman is an open-source, self-hosted translation management system built for teams that want to manage their localization work through an AI agent — not a web dashboard. You connect Turjuman's MCP server to Claude Code (or any MCP-compatible client), and your agent handles projects, locales, keys, translations, and team access in natural language. A thin developer CLI handles the deterministic work — pulling and pushing locale files in your repo and CI. The whole system runs serverless on AWS Lambda and DynamoDB: no always-on servers, no Cognito, and effectively free for small teams.

## How it fits together

```
  Claude Code / MCP client
        │  Streamable HTTP + API key
        ▼
  MCP Lambda Function URL  ──► DynamoDB (single table)
                                    ▲
  REST API Lambda Function URL ─────┘
        ▲  REST + API key
        │
  turjuman CLI / CI pipeline
```

Your MCP client and your developer CLI both authenticate with a bearer API key. The MCP server exposes the full management surface — create projects, add locales, fill translations, run QA — while the REST API backs the CLI's file-sync commands. Both Lambda functions are stateless; DynamoDB is the single source of truth. Because the connected LLM does the translating itself, there is no machine-translation engine to configure or pay for.
