This guide gets you a fully working Kalem stack running on your machine in minutes. Instead of deploying to AWS, you run DynamoDB Local in Docker and start the MCP and API servers directly from the repo. No cloud account, no credentials, no cost — just Node.js and Docker.

<Note>
  The API key printed by the setup script determines both your identity and your permissions. Whoever holds it has full OWNER access. Store it somewhere safe and keep it out of version control.
</Note>

## Prerequisites

- Node.js 20 or later
- Docker (for DynamoDB Local)
- The Kalem repo cloned locally

## Steps

<Steps>
  <Step title="Install dependencies and build">
    Clone the repo (if you haven't already) and run the standard install + build:

    ```bash
    npm install && npm run build
    ```

    This compiles all workspace packages — `mcp-server`, `api`, `cli`, and `core` — into their `dist/` directories.
  </Step>

  <Step title="Start DynamoDB Local">
    Start a local DynamoDB instance on port 8000 using the official Docker image:

    ```bash
    docker run -d --rm -p 8000:8000 amazon/dynamodb-local
    ```

    The `--rm` flag removes the container when it stops. Your data lives only in memory for the duration of the container's lifetime — that's fine for local evaluation.
  </Step>

  <Step title="Create the table and your first owner">
    Run the setup script to create the DynamoDB table and bootstrap the first OWNER user:

    ```bash
    node scripts/dev-setup.mjs you@example.com "Your Name"
    ```

    The script prints your API key **once**. Copy it now — it is not stored in plaintext anywhere and cannot be recovered.

    ```
    Table created: Kalem
    Owner created: you@example.com
    API key: km_live_xxxxxxxxxxxxxxxxxxxx

    Store this now — it is shown only once.
    ```

    If you re-run the script after the org already has users, it skips the bootstrap step safely and tells you to reuse your existing key.
  </Step>

  <Step title="Start the MCP server and API server">
    Open two terminal windows. In each one, export the environment variables that point the servers at DynamoDB Local:

    ```bash
    export AWS_ENDPOINT_URL_DYNAMODB=http://localhost:8000 KALEM_TABLE=Kalem
    export AWS_ACCESS_KEY_ID=local AWS_SECRET_ACCESS_KEY=local AWS_REGION=us-east-1
    ```

    **Terminal 1 — MCP server** (listens on port 3000 by default):

    ```bash
    node packages/mcp-server/dist/local.js
    ```

    **Terminal 2 — REST API** (listens on port 3001 by default):

    ```bash
    node packages/api/dist/local.js
    ```

    Both servers log their listening address on startup. Leave them running.
  </Step>

  <Step title="Connect your MCP client">
    Add Kalem to your MCP client configuration. For Claude Code (`.mcp.json` in your project root):

    ```json
    {
      "mcpServers": {
        "kalem": {
          "type": "http",
          "url": "http://localhost:3000/",
          "headers": {
            "Authorization": "Bearer <your-api-key>"
          }
        }
      }
    }
    ```

    Replace `<your-api-key>` with the key printed in Step 3. Restart your MCP client to pick up the new config.

    <Note>
      Keep your API key out of version control. If `.mcp.json` is committed, reference the key via an environment variable or a secrets manager rather than pasting the literal value. Revoke any leaked key immediately using the `revoke_api_key` MCP tool.
    </Note>
  </Step>

  <Step title="Try it">
    With the MCP server connected, ask your agent to get started:

    > *"Create a project called Web App with base locale `en`. Add `fr` and `es` locales. Then translate all the keys into French and Spanish."*

    Kalem's MCP tools handle the project setup, and the connected LLM translates the keys directly — no separate translation engine needed.

    You can also ask things like:
    - *"What projects do I have?"*
    - *"Add a key called `checkout.empty_cart` with the English value `Your cart is empty.`"*
    - *"Run QA on the French locale and show me any issues."*
  </Step>
</Steps>
